# Project Documentation Prompt

## Analysis Requirements
- Identify key stakeholders and their needs.
- Define project objectives and success criteria.
- Conduct a risk analysis and mitigation strategy.

## Documentation Structure
1. **Introduction**
   - Overview of the project
   - Purpose and scope
2. **System Architecture**
   - Overview of system components
   - Relationships between components
   - Include mermaid diagrams for visual representation
3. **Class Diagrams**
   - Define classes and relationships
   - Use UML notation
4. **Sequence Diagrams**
   - Detail interactions between objects over time
   - Use UML notation
5. **Flowcharts**
   - Outline processes and workflows
6. **State Diagrams**
   - Show states of an object and transitions
7. **Entity-Relationship Diagram (ERD)**
   - Define data entities and relationships
8. **Data Flow Diagrams**
   - Illustrate how data moves through the system
9. **Additional Diagram Recommendations**
   - Timeline diagrams for project phases
   - Git workflow diagrams for version control
   - Deployment pipeline diagrams for deployment strategies

## Conclusion
- Summary of documentation importance
- Next steps for documentation and approval
